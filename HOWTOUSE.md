## 🔄 基本的な更新の流れ

### 1. ローカル環境でのプレビュー
```bash
# プロジェクトディレクトリに移動
cd /Users/hgotoh/code/poclab.github.io

# ローカルサーバーを起動
bundle exec jekyll serve --port 4002 --host localhost

# ブラウザで http://localhost:4002 を開いて確認
```

### 2. ファイル編集 → プレビュー → 公開
1. **ファイルを編集**
2. **ブラウザで変更を確認**（自動更新されます）
3. **満足したらGitHubにプッシュ**
4. **GitHub Pagesで自動公開**

## 📄 論文・発表の追加

### 新しい論文を追加する

**場所**: `_publications/` ディレクトリ

#### 1. 新しいファイルを作成
```bash
# ファイル名は日付-論文タイトル.md の形式

```

```markdown

collection: publications
category: manuscripts
permalink: /publication/2025-07-26-new-paper

date: 2025-07-26

## Abstract
英語の要約

```

## 👥 メンバー情報の更新

研究室メンバーの情報は、`_members`ディレクトリのMarkdownファイルで管理されます。この方法により、HTMLを知らなくても簡単にメンバー情報を更新できます。

### 🆕 新しいメンバーの追加

#### 手順
1. `_members`ディレクトリに新しいMarkdownファイルを作成
2. ファイル名は「姓-名.md」形式（例：`yamada-hanako.md`）
3. ファイルの先頭にYAMLフロントマターを記入
4. サイトを再構築すると自動的にメンバーページに表示

#### 作成場所
```
_members/新メンバー名.md
```

### 🎯 重要な注意事項

#### ファイル名のルール
- **必須**: ローマ字で「姓-名.md」
- **例**: `tanaka-taro.md`, `yamada-hanako.md`
- **注意**: スペースや特殊文字は使用不可

#### YAMLフロントマターの必須項目
```yaml
---
title: "日本語氏名"          # 必須
position: "学年・職位"      # 必須
level: "faculty/graduate/undergraduate"  # 必須（自動分類用）
has_detail_page: true/false  # 詳細ページの有無（true=リンク付き、false=名前のみ）
research_theme: "研究テーマ"  # 学生の場合は必須（詳細ページありの場合）
---
```

#### 詳細ページの有無による管理
- `has_detail_page: true`: 名前がリンクになり、個別プロフィールページが表示される
- `has_detail_page: false`: 名前のみ表示、リンクなし（シンプルな一覧表示）

#### レベル分類の説明
- `faculty`: 教員（教授、准教授、助教など）
- `graduate`: 大学院生（博士、修士）
- `undergraduate`: 学部生
- `alumni`: 修了生・卒業生（アーカイブ用）

### 🔧 緊急時の対応

#### サイトが表示されない場合
1. YAMLフロントマターの文法エラーを確認
2. ファイル名に特殊文字がないか確認
3. 必須項目（title, position, level）が記入されているか確認

#### 文字化けする場合
1. ファイルがUTF-8で保存されているか確認
2. VS Codeの右下でエンコーディングを確認

#### 古い情報が表示される場合
1. ブラウザのキャッシュをクリア（Ctrl+F5 / Cmd+Shift+R）
2. Jekyllサーバーを再起動

## 🔧 よくある問題と解決方法

### 1. サーバーが起動しない

#### ポートが使用中の場合
```bash
# 別のポートを使用
bundle exec jekyll serve --port 4003 --host localhost
```

#### 依存関係の問題
```bash
# Gemfile.lockを削除して再インストール
rm Gemfile.lock
bundle install
```

### 2. 画像が表示されない

#### 画像の配置場所
- 画像は `images/` ディレクトリに配置
- Markdownでの参照: `![alt text](/images/filename.png)`
